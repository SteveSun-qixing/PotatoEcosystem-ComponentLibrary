# 仓库重建策略

## 1. 重建步骤

1. 清理旧实现目录（保留必要历史说明）。
2. 建立新目录结构与工作区配置。
3. 引入 Ark UI、Style Dictionary、测试工具链。
4. 建立统一 lint/type/test/build 发布门禁。

## 2. 分支策略

- 主开发分支：`codex/ark-ui-greenfield-docs`（当前）
- 后续功能分支：`codex/ark-ui-<feature>`
- 提交规范：按阶段合并，禁止无文档需求的直接编码。

## 3. 依赖策略

- 主包管理：pnpm workspace。
- 生态内部依赖：使用 `workspace:*` 协议。
- 外部依赖：锁定主版本，定期批量升级。
