# 组件状态模型与交互规范

## 1. 状态机

每个交互组件至少支持：

- default
- hover
- focus-visible
- active/pressed
- disabled
- invalid
- loading

## 2. 状态来源

- 用户输入事件
- 表单上下文
- 异步请求生命周期
- 外部可控 props

## 3. 交互输出

- `data-state` 标识主状态。
- `aria-*` 同步无障碍语义。
- 统一事件总线输出变更事件。

## 4. 规范

- 禁止状态歧义（如 disabled 同时可交互）。
- 禁止视觉状态和语义状态脱节。
- 强制键盘路径等价于鼠标路径。
