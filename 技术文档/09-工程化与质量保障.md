# 工程化与质量保障

## 1. 工具链

- TypeScript 严格模式
- ESLint + Prettier
- Vitest + Playwright（必要时）
- Coverage 报告
- Changeset / Changelog

## 2. 质量门禁

- `lint` 通过
- `type-check` 通过
- `test` 通过
- `build` 通过
- 关键包产物体积检查

## 3. 测试分层

- 单元测试：状态机、hook、工具函数
- 组件测试：交互、无障碍、主题注入
- 合同测试：SDK/Foundation 接口契约
- 集成测试：组件库运行时与发布流程回归
