# 多语言与配置表实现

## 1. i18n 方案

- 开发阶段：使用语义 key + 开发词汇表。
- 构建阶段：编码映射替换（遵循系统词汇编码）。
- 运行阶段：按 locale 加载语言包，支持 fallback。

## 2. 配置表体系

- `config/default`：系统默认配置。
- `config/app`：应用级覆盖。
- `config/user`：用户偏好。
- `config/runtime`：会话级临时配置。

## 3. 组件接入要求

- UI 文案只能来自 i18n API。
- 行为阈值和开关从配置表读取。
- 配置变更必须触发订阅更新与日志记录。
