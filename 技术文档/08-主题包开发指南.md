# 主题包开发指南

## 主题包概述

主题包是薯片生态中独立的视觉设计产品,为组件库提供完整的样式实现。本指南说明如何开发、测试和发布主题包。

## 开发环境搭建

### 使用脚手架创建主题包

```bash
# 安装脚手架工具
npm install -g @chips-ui/create-theme

# 创建新主题包
create-chips-theme my-theme

# 进入项目目录
cd my-theme

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

### 主题包项目结构


## 创建主题包

### 第一步：配置主题元数据

```json
// package.json
{
  "name": "@chips-theme/my-theme",
  "version": "1.0.0",
  "description": "我的主题包描述",
  "author": "Your Name",
  "license": "MIT",
  "chipsTheme": {
    "id": "my-theme",
    "displayName": "我的主题",
    "category": "light",
    "tags": ["现代", "简洁", "商务"],
    "preview": "./preview.png",
    "compatibility": {
      "minVersion": "1.0.0",
      "maxVersion": "2.x"
    }
  },
  "main": "dist/index.css",
  "files": [
    "dist",
    "src",
    "preview.png",
    "screenshot"
  ]
}
```

## 开发和测试

### 本地预览

开发服务器会启动一个预览页面,展示所有组件在主题下的效果:

```bash
npm run dev
```

访问 http://localhost:3000 查看主题效果。

### 测试主题完整性

```bash
# 运行主题验证
npm run validate

# 检查必需的样式是否都已定义
npm run check-coverage
```

### 生成预览图

```bash
# 自动生成主题预览图
npm run generate-preview

# 生成所有组件截图
npm run generate-screenshots
```

## 主题包最佳实践

### 保持一致性

- 使用统一的颜色系统
- 保持间距和尺寸的规律性
- 统一圆角和阴影风格
- 协调动画速度和缓动函数

### 可访问性

- 确保文本对比度符合WCAG标准
- 提供清晰的焦点指示
- 支持高对比度模式
- 考虑色盲用户

### 性能优化

- 避免过度使用阴影和滤镜
- 优化动画性能
- 压缩CSS文件
- 使用CSS变量减少重复代码

### 文档完善

编写详细的主题包文档:

```markdown
# 我的主题包

## 设计理念

这个主题包采用现代简洁的设计风格...

## 颜色方案

- 主色: #1890ff (蓝色)
- 成功色: #52c41a (绿色)
- 警告色: #faad14 (橙色)
- 错误色: #f5222d (红色)

## 使用方法

\`\`\`typescript
import '@chips-theme/my-theme';

<ThemeProvider theme="my-theme">
  <App />

\`\`\`

## 自定义

可以通过覆盖CSS变量进行自定义...

- Chrome 90+

```

## 总结

主题包开发需要综合考虑设计、技术和用户体验。通过定义清晰的设计tokens、编写完整的组件样式、进行充分的测试,可以创建高质量的主题包。遵循最佳实践,确保主题的一致性、可访问性和性能,为用户提供优秀的视觉体验。
