# 现状评估与重建决议

## 1. 现状评估结论

于 2026-02-10 对旧版组件库进行基线检查：

- `npm run test:run`：可通过（42 个测试文件，1166 个测试点）。
- `npm run build`：失败，出现大量 TypeScript 严格类型错误（`exactOptionalPropertyTypes`、未使用变量、类型不兼容等）。
- 测试日志存在上下文注入告警（例如 i18n context 未注入）。

## 2. 风险判断

旧版代码存在以下结构性问题：

1. 构建不可发布：测试通过但产物构建失败，无法作为生态稳定基座。
2. 规范一致性不足：严格类型、上下文依赖、工程门禁未形成闭环。
3. 架构方向偏差：旧版以 Vue 组件库为主，和本轮 Ark UI + Style Dictionary 的目标路线不一致。

## 3. 决议

- 决议 A：旧版实现不作为继续演进基线。
- 决议 B：采用“新分支 + 新架构 + 新手册 + 新计划”推进。
- 决议 C：开发过程优先优化架构，不做兼容性负担，不做历史包袱迁移。

## 4. 处置原则

- 可复用的是“规范与领域知识”，不是“旧实现代码”。
- 旧仓库内容仅作为历史参考，后续开发以新文档和新目录为准。
- 所有新实现必须先通过手册定义再进入编码。
