# 非功能需求与质量红线

## 1. 质量红线

1. 构建必须通过（类型检查 + 打包）。
2. 单元测试、集成测试、可访问性测试必须入门禁。
3. 任何硬编码主题样式或硬编码文案，视为阻断缺陷。
4. 关键组件出现键盘不可达/焦点丢失，视为阻断缺陷。

## 2. 性能目标

- 基础组件首次渲染延迟可控。
- 主题切换需支持局部刷新，避免整树重渲染。
- token 注入和样式回退流程可缓存。

## 3. 安全目标

- 文本渲染防注入。
- 样式变量白名单校验，防止非法主题注入。
- 资源路径解析遵守路径安全约束。

## 4. 可维护性目标

- 文档先行：需求、技术、计划与实现保持一致。
- 分层清晰：schema、runtime、component、integration 各司其职。
- 高内聚低耦合：组件间不共享隐式状态。
