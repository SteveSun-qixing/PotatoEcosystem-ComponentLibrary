# 范围边界与不做事项

## 1. 本期范围（In Scope）

- 组件库技术底座重建（Ark UI + Style Dictionary + tokens schema）。
- 主题分层与回退机制。
- P0/P1 组件优先级清单与基础交互协议。
- 与 Core / Foundation / SDK 的集成边界定义。
- i18n 编码体系接入与配置表规范。

## 2. 非本期范围（Out of Scope）

- 旧版本 API 兼容层。
- 为旧项目提供自动迁移脚本。
- 一次性实现全部主题包的视觉高度还原。
- 与未冻结规范的外部模块做深度耦合开发。

## 3. 开发原则

1. 优先架构正确，其次功能覆盖，最后视觉主题扩展。
2. 先定义协议，再实现组件，最后做主题和联调。
3. 新版本不背历史兼容包袱，直接面向生态长期演进。
