## 2. 关键约束提炼

1. 必须坚持中心路由原则，组件库不做跨层直连。
2. 组件库属于无样式逻辑层，视觉由主题包提供。
3. 主题支持分层覆盖和逐级回退，且是全量 token 回退。
4. 多语言必须编码化、配置化，禁止硬编码文本。
5. 工程质量必须包含构建、类型、测试、日志、错误处理闭环。

## 3. 设计决策映射

- 采用 Ark UI 承载交互逻辑和可访问性原语。
- 采用 Style Dictionary 管理 token 协议和主题产物。
- 采用阶段化开发计划，先协议后实现，先底座后组件。
- 不做旧版兼容迁移，直接面向首版可持续架构。
