# 项目背景与重建目标

## 1. 背景

薯片生态是“微内核 + 公共基础层 + SDK + 应用/插件”的分层系统。组件库属于生态前端能力核心，承担统一交互结构、可访问性、主题对接和多语言承载职责。

根据生态设计原稿与共用规范，组件库必须满足以下前提：

- 只承载功能逻辑、结构和布局，不固化视觉皮肤。
- 视觉样式、动效参数、体积层级均由主题包注入。
- 所有生态软件共享同一套组件协议，保证跨软件协同。
- 所有模块通信遵循中心路由原则，不做跨模块直连。

## 2. 重建目标

本轮是**从零新建**，不是旧项目重构。目标如下：

1. 基于 Ark UI 构建 headless 组件能力层。
2. 基于 Style Dictionary 构建多主题、多层级、可回退主题系统。
3. 建立符合生态规范的 i18n、配置化、日志、测试、发布体系。
4. 完成与 `Chips-core`、`Chips-Foundation`、`Chips-SDK` 的接口对齐。
5. 为后续主题包（液态玻璃、Material 3、Fluent、Bento、Neobrutalism、Atlassian、Linear、Notion 等）提供稳定协议。

## 3. 成功标准

- 组件库可独立构建与测试，具备可发布包结构。
- 核心 P0 组件在“无样式 + 多主题注入”模式下稳定运行。
- 主题层级回退机制可按“本层未设置时逐级向上查找”生效。
- 所有文本遵循多语言编码规范，不允许硬编码 UI 文案。
- 与 SDK / Foundation 的对接有可执行联调清单。
