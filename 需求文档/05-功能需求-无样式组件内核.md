# 功能需求：无样式组件内核

## 1. 目标

构建仅负责“语义结构 + 状态逻辑 + 交互行为 + 可访问性”的组件层，不绑定任何品牌样式。

## 2. 组件分层

- Primitive 层：Ark UI 原语封装。
- Pattern 层：薯片语义组件（Button、Input、Dialog 等）。
- Scene 层：卡片渲染与布局场景组件。

## 3. 强制能力

1. 状态协议：idle / hover / focus / pressed / disabled / loading / invalid。
2. 数据属性：统一输出 `data-scope`、`data-part`、`data-state`。
3. 可访问性：键盘导航、焦点管理、ARIA 属性完整。
4. 结构稳定：主题不可破坏 HTML 结构和布局契约。

## 4. 组件优先级

- P0：Button、Input、Textarea、Select、Checkbox、Radio、Switch、Dialog、Tooltip、Popover、Menu、Tabs、Form。
- P1：Card Wrapper、Layout、Grid/Flex、Badge/Tag、Pagination、Skeleton、Message。
- P2：复杂数据展示与高阶交互组件。
