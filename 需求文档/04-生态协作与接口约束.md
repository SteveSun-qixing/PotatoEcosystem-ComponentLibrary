# 生态协作与接口约束

## 1. 分层协作定位

组件库在生态中的定位：

- 上接：生态应用 / 第三方应用。
- 横向：插件前端实现。
- 下接：SDK API、Foundation 通用能力。
- 核心约束：跨模块请求走内核路由协议。

## 2. 接口约束

依据 `生态共用/05-前端接口标准.md`：

- 组件需实现明确生命周期（initialize / mount / update / unmount / destroy）。
- 渲染选项需支持 `mode`、`theme`、`readonly`、`locale` 等标准字段。
- 主题系统接口必须支持组件粒度应用。
- 无障碍、响应式、错误边界均属于标准接口范围。

## 3. 协作约束

依据 `生态共用/14-模块联动协作规范.md`：

- 应用层通过 SDK 调用组件能力。
- 组件库不得跳过 SDK 直接绑定业务模块。
- 主题变量传递必须遵守层级覆盖和继承规则。
- 模块注册信息必须可发现、可追踪。

## 4. 依赖约束

依据 `生态共用/12-依赖管理指南.md`：

- 使用 workspace 协议管理生态内依赖。
- 构建顺序遵循：Core -> Foundation -> SDK -> ComponentLibrary -> Apps。
- 锁定关键依赖版本，避免无控制漂移。
